# v0.5.0 - Local Enterprise Operations

This release consolidates deterministic-agent-system as a locally operable supervised agent runtime.

## Highlights

- Added protected operational readiness endpoint: GET /ready.
- Added protected operational metrics endpoint: GET /metrics.
- Added configurable HTTP rate limiting for WhatsApp webhook and ops endpoints.
- Added signed WhatsApp webhook POST validation when WHATSAPP_APP_SECRET is configured.
- Added persistent conversation evidence endpoint.
- Added persistent conversation event log endpoint.
- Added WhatsApp handoff list and close operations.
- Added pending handoff alert script.
- Added operational snapshot script combining readiness, metrics, handoffs, and optional backup.
- Added SQLite backup script with SHA-256 manifest.
- Added local CLI wrapper for operational scripts.
- Added live pilot operations runbook.
- Expanded contractual baseline to 191 tests.

## Local CLI

The local CLI wraps existing PowerShell operations without duplicating logic:

    npm run cli -- help
    npm run cli -- smoke
    npm run cli -- snapshot -SkipBackup
    npm run cli -- check-handoffs
    npm run cli -- backup -KeepLast 20

The package also includes a local bin entry:

    det-agent

The package remains private in this release and is not published to npm.

## Operational endpoints

- GET /health
- GET /ready
- GET /metrics
- GET /whatsapp/handoffs
- POST /whatsapp/handoffs/:handoffId/close
- GET /whatsapp/conversations/:customerId/evidence
- GET /whatsapp/conversations/:customerId/events
- GET /webhooks/whatsapp
- POST /webhooks/whatsapp

Ops endpoints are protected with x-ops-token.
